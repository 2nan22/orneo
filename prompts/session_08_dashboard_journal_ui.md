## 꼭지 1: 라이프 캐피털 대시보드 화면

보고서 와이어프레임 구현:

```
┌──────────────────────────────┐
│ 이번 주 라이프 캐피털 점수   │
│        [점수 게이지]  78     │
│ 자산 안정성 / 목표 진척 / 루틴│
├──────────────────────────────┤
│ 오늘의 핵심 질문             │
│ "지금 매수보다 현금 확보가?" │
├──────────────────────────────┤
│ 오늘 할 행동 3개             │
│ ○ 실거래가 변화 확인         │
│ ○ 투자 가설 일지 작성        │
│ ○ K-MOOC 20분 학습           │
└──────────────────────────────┘
```

**컴포넌트**
- `CapitalScoreGauge`: 0~100 점수 원형 게이지 (SVG 또는 CSS conic-gradient)
- `ScoreBreakdown`: 세 축 점수 프로그레스 바 (Point Color #00C2A8 사용)
- `TodayActions`: 체크 가능한 행동 목록 (체크 시 줄긋기 + 로컬 상태)
- `KeyQuestion`: 오늘의 핵심 질문 카드 (강조 타이포)

**완료 기준**
- [ ] `/api/v1/dashboard/` API 데이터 연동 확인
- [ ] 행동 체크 시 UI 즉시 반영 (낙관적 업데이트)
- [ ] 모바일 375px 기준 반응형 확인

**커밋**
```
feat(frontend): 라이프 캐피털 대시보드 화면 구현
```

## 꼭지 2: 의사결정 일지 목록 화면

**화면 구성** (`/journal`)
- 상단: 카테고리 탭 필터 (투자 / 주거 / 학습 / 루틴 / 전체)
- 목록: JournalCard 컴포넌트
  - 카테고리 배지 (색상 구분)
  - 제목 + AI 요약 미리보기 (요약 생성 중이면 로딩 스피너)

## 꼭지 3: 일지 작성 & 복기 화면

**일지 작성 화면** (`/journal/new`)
- 카테고리 선택 버튼
- 제목 Input
- 내용 Textarea (판단의 이유·근거)
- 감정 점수 선택 (😔 😕 😐 😊 😄 1~5)
- 관련 목표 연결 드롭다운
- 저장 버튼 → 저장 후 "AI가 요약을 생성 중입니다..." 토스트 표시

**복기 모달** (일지 목록/상세에서 접근)
- 기존 일지 내용 + AI 요약 요약 표시 (읽기 전용)
- 복기 메모 Textarea
- 저장 → `PATCH /api/v1/journals/{id}/review/` 호출

**완료 기준**
- [ ] 일지 작성 → 목록에서 카드 확인 플로우 동작
- [ ] 복기 저장 후 "복기 필요" 배지 → "복기 완료" 배지로 변경 확인
- [ ] AI 요약 생성 후 목록 카드 자동 업데이트 (polling 5초 간격)

**커밋**
```
feat(frontend): 일지 작성·복기 화면 구현
```
